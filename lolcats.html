LoLCATs: Linearizing Large Language Models

Low-rank linear conversion via attention transfer — how to turn a quadratic Transformer into a subquadratic model using 0.2% of its parameters.

The Quadratic Attention Problem

Transformer language models owe their power to softmax attention, which lets every token attend to every other token in the sequence. But this comes at a cost: computing attention requires $O(n^2)$ time and memory in the sequence length $n$. For long contexts — 128K tokens and beyond — this quadratic scaling becomes the dominant bottleneck.

Linear attention replaces the softmax kernel with learnable feature maps, reducing the cost to $O(n)$. The problem? Directly substituting linear attention into a pretrained Transformer destroys model quality. Prior methods that retrain from scratch require billions of tokens and massive compute.

LoLCATs (Low-rank Linear Conversion via Attention Transfer) solves this with a two-stage approach: first train linear attention feature maps to mimic softmax attention outputs, then apply low-rank adaptation to recover any remaining quality. The result: linearized 8B, 70B, and 405B models using only 0.2% of parameters and 40M training tokens.

Softmax vs. Linear Attention

Standard softmax attention computes the output for token $n$ as a weighted sum over all prior tokens:

$$y_n = \sum_{i=1}^{n} \frac{\exp\!\left(q_n^\top k_i / \sqrt{d}\right)}{\displaystyle\sum_{j=1}^{n} \exp\!\left(q_n^\top k_j / \sqrt{d}\right)} \cdot v_i$$

where $q_n, k_i \in \mathbb{R}^d$ are query and key vectors and $v_i \in \mathbb{R}^{d_v}$ is the value vector. The softmax normalizes the attention weights so they sum to one. Computing this requires materializing an $n \times n$ attention matrix — hence the quadratic cost.

The Linear Attention Alternative

Linear attention replaces the exponential kernel with learnable feature maps $\phi_q$ and $\phi_k$:

$$\hat{y}_n = \frac{\displaystyle\sum_{i=1}^{n} \phi_q(q_n)^\top \phi_k(k_i) \cdot v_i}{\displaystyle\sum_{i=1}^{n} \phi_q(q_n)^\top \phi_k(k_i)}$$

The key insight: by the associativity of matrix multiplication, we can rewrite this using cumulative sums instead of pairwise dot products. Define the KV state $S_n = \sum_{i=1}^{n} \phi_k(k_i) v_i^\top$ and normalization $z_n = \sum_{i=1}^{n} \phi_k(k_i)$. Then:

$$\hat{y}_n = \frac{\phi_q(q_n)^\top S_n}{\phi_q(q_n)^\top z_n}$$

Since $S_n$ and $z_n$ can be updated incrementally ($S_n = S_{n-1} + \phi_k(k_n) v_n^\top$), each new token only requires $O(d \cdot d')$ work — linear in sequence length, not quadratic.

Stage 1: Attention Transfer

The core idea of LoLCATs is to train feature maps $\phi_q$ and $\phi_k$ so that the linear attention output matches the softmax attention output as closely as possible. This is formulated as an MSE regression problem, layer by layer.

The Training Objective

For a model with $M$ attention layers and $H$ heads per layer, we minimize:

$$\ell_{\text{MSE}} = \frac{1}{MH} \sum_{m=1}^{M} \sum_{h=1}^{H} \left\| y^{(h,m)} - \hat{y}^{(h,m)} \right\|_2^2$$

where $y^{(h,m)}$ is the softmax attention output (computed via FlashAttention) and $\hat{y}^{(h,m)}$ is the linear attention output for head $h$ in layer $m$. Only the feature map parameters are trained; all original model weights stay frozen.

Feature Map Architecture

Each feature map is parameterized as a learned affine projection followed by a nonlinearity $f$, applied after rotary positional embeddings (RoPE):

$$\phi_q(q_n) = f\!\left(\text{RoPE}(q_n)\, \widetilde{W}_q + \widetilde{b}_q\right)$$
$$\phi_k(k_i) = f\!\left(\text{RoPE}(k_i)\, \widetilde{W}_k + \widetilde{b}_k\right)$$

where $\widetilde{W}_q, \widetilde{W}_k \in \mathbb{R}^{d \times d'}$ and $\widetilde{b}_q, \widetilde{b}_k \in \mathbb{R}^{d'}$. This adds only 16.8M trainable parameters for an 8B model — roughly 0.2% of total parameters.

Two-Stage LoLCATs Pipeline

Hybrid: Sliding Window + Linear Attention

Pure linear attention struggles with local, spiky attention patterns — cases where a token attends heavily to a few nearby tokens. LoLCATs addresses this with a hybrid architecture that uses exact softmax attention within a local window and linear attention for everything beyond it.

The Hybrid Formulation

For a window size $w$, the output at position $n$ combines a local softmax window with a global linear component:

$$\hat{y}_n = \frac{\displaystyle\sum_{i=n-w+1}^{n} \gamma \cdot e^{q_n^\top k_i/\sqrt{d}\, - \,c_n} \cdot v_i \;+\; \phi_q(q_n)^\top \!\sum_{j=1}^{n-w} \phi_k(k_j)\, v_j^\top}{\displaystyle\sum_{i=n-w+1}^{n} \gamma \cdot e^{q_n^\top k_i/\sqrt{d}\, - \,c_n} \;+\; \phi_q(q_n)^\top \!\sum_{j=1}^{n-w} \phi_k(k_j)}$$

where $c_n$ is a stabilizing constant and $\gamma$ is a learned gate. The local window handles precise nearby attention with $O(w^2 d)$ cost, while the linear component captures long-range dependencies in $O(d \cdot d')$. With a default window of $w = 64$, the total cost remains subquadratic.

Generation Complexity

At inference time, generating each new token costs $O(w^2 d + d \cdot d')$ in both time and memory — compared to $O(n \cdot d)$ for full softmax attention, which grows with every token generated.

Hybrid Attention Architecture

Stage 2: Low-Rank Adjusting

After swapping in the trained linear attention layers, some quality gap remains. LoLCATs closes this with LoRA (Low-Rank Adaptation) applied to the attention projection matrices.

LoRA Weight Update

Each attention projection weight $W$ is updated with a low-rank correction:

$$W \leftarrow W + \Delta W = W + BA$$

where $B \in \mathbb{R}^{d \times r}$ and $A \in \mathbb{R}^{r \times d}$ with rank $r \ll d$. With the default $r = 8$, this adds fewer than 0.09% additional parameters. The LoRA weights are trained on standard next-token prediction loss with the linearized attention layers frozen.

Why Two Stages?

The key insight is that attention transfer alone gets you most of the way — it ensures the linear attention layers produce outputs close to what softmax would have produced. But small approximation errors compound across layers. LoRA provides a lightweight correction that lets the rest of the network adjust to these small differences, without retraining from scratch.

Training Perplexity

Scaling: Block-wise Attention Transfer

Training all layers jointly works for 8B models, but at 70B+ scale a problem emerges: later layers receive far worse approximations because errors compound through the network during joint training. Later layers can exhibit 200$\times$ higher MSE than earlier ones.

Block-wise MSE

LoLCATs solves this by dividing the $M$ layers into blocks of $b$ layers and training each block independently with its own supervision signal:

$$\ell_{\text{MSE}}^{\text{block}} = \frac{1}{bH} \sum_{m=i}^{i+b} \sum_{h=1}^{H} \left\| y^{(h,m)} - \hat{y}^{(h,m)} \right\|_2^2$$

Smaller block sizes $b$ improve quality but increase training time by a factor of $M/b$. The block-wise approach requires precomputing hidden states at block boundaries — but these can be computed once and reused, and blocks can be trained in parallel across GPUs.

This is what enables linearization at scale: the 405B Llama 3.1 model was linearized with attention transfer in just 5 hours on 14 H100 GPUs, followed by 16 hours of LoRA finetuning.

Results

LoLCATs produces the first viable linearizations of 70B and 405B parameter models, while also setting new state-of-the-art for 8B linearization — all at a fraction of prior training cost.

Benchmark Comparison

5-shot MMLU Accuracy

On Llama 3 8B, LoLCATs achieves 73.1 MMLU vs. 66.6 for the original Transformer — a 20+ point improvement over prior linearization methods, while using 0.2% of parameters and 0.4% of training tokens. On the 405B model, LoLCATs closes 78.1% of the quality gap to the original Transformer.

Efficiency Gains

The linearized models provide concrete inference speedups: 3$\times$ higher throughput than FlashAttention-2 and the ability to use 64$\times$ larger batch sizes on the same hardware, since the KV cache no longer grows linearly with sequence length.
